How This Page Works

This section is a small tutorial: it uses the LogClient and the functions API to drive the panels above. You can copy these patterns into your own experiment UIs when you add a custom ui/ and set entry_point in the README frontmatter.

1. Create a LogClient

Either from the current page (experiment is inferred from the URL path /exp/<name>/... or ?exp=<name>), or by passing options explicitly: 

import { LogClient } from "/static/logclient.js";

// Option A: auto-detect experiment from URL
const client = LogClient.fromCurrentPage();

// Option B: explicit (e.g. when not under /exp/<name>/)
const client = new LogClient({
  baseUrl: window.location.origin,
  experiment: "default"
});

2. Where each panel gets its data

  • Live Stats (log count, students) — client.getLogOptions() → { students, trials, log_count }. Backed by GET /exp/<name>/log-options.
  • Live Stats (function count) — fetch("/exp/<name>/functions") → object of function names to { signature, doc, nolog, noregcheck, ratelimit }.
  • Recent Logs — client.getLogs({ n: 10 }) → { logs: [...] }. Backed by GET /exp/<name>/logs?n=10.
  • Calls per Function — client.getAllLogs({}) returns every log (auto-paginated); this page then counts by func_name and renders the bars.

3. Log entry shape

Every item in logs has the same shape. Use it when building visualizations or tables: 

{
  "id": 1,
  "ts": "2025-03-11T12:00:00Z",
  "student_id": "s001",
  "experiment": "default",
  "trial": "run-1",
  "func_name": "square",
  "args": [7],
  "result": 49,
  "error": null
}

4. LogClient methods (quick reference)

// One page of logs (default n=100, order=latest)
const { logs } = await client.getLogs({ n: 50, order: "earliest" });

// Filter by student, function, or time range
const { logs } = await client.getLogs({
  studentId: "s001",
  funcName: "square",
  startTime: "2025-03-01T00:00:00Z",
  endTime: "2025-03-11T23:59:59Z"
});

// All logs matching filters (auto-paginates; use with care on large datasets)
const allLogs = await client.getAllLogs({ funcName: "gradient_step" });

// Summary for dropdowns and counts
const opts = await client.getLogOptions();
// opts.students, opts.trials, opts.log_count

5. Calling functions from the browser

The LogClient is read-only (logs and options). To invoke experiment functions from JS, use the RPCClient — it mirrors the Python client's API with dynamic dispatch, error handling, and convenience methods: 

import { RPCClient } from "/static/rpcclient.js";

// Auto-detect experiment from URL; studentId is required
const client = RPCClient.fromCurrentPage({ studentId: "s001" });

// Dynamic dispatch (lazy discovery on first call)
const result = await client.square(7);  // 49

// Explicit call with per-call trial override
const r2 = await client.call("add", 3, 5, { trial: "demo" });

// Explore available functions
console.log(await client.help());

// Check registration and fetch logs
console.log(await client.isRegistered());       // true/false
console.log(await client.fetchLogs({ n: 5 }));  // latest 5 logs

6. Build Admin Dashboards & Auth Gates

LEAP2 naturally supports secure Admin dashboarding. Use the /api/auth-status endpoint to verify access, and safely summon the platform's global login modal via window.LEAP.showLogin() if they are unauthenticated. 

import { AdminClient } from "/static/adminclient.js";

fetch("/api/auth-status", { credentials: "same-origin" })
  .then(r => r.json())
  .then(async d => {
    if (d.admin) {
      // User is a verified Admin
      const adminClient = AdminClient.fromCurrentPage();
      await adminClient.addStudent("demo1", "Demo User");
    } else {
      // Summon global login modal natively
      if (window.LEAP && window.LEAP.showLogin) {
        window.LEAP.showLogin(() => window.location.reload());
      }
    }
  });

7. Publishing & Community Registry

LEAP2 experiments are designed to be shared. Distribution is handled via Git and our CLI, allowing you to easily pull remote experiments from the global registry directly into your local lab: 

# Discover interactive experiments from the community
leap discover --tag optimization

# Install an experiment directly from a Git URL into your lab
leap add https://github.com/user/cool-lab.git

# Publish your own local experiment to the registry for review
leap publish my-experiment
